Beispielszenario: Vertriebsprozessmodellierung zur Managementunterstützung

Visualisierung der Zusammenhänge zwischen Support-Bearbeitungsprozessen, beteiligten Komponenten und Auftraggebern der Dataport-Kunden.

Dataport interne Kunden (im Folgenden User genannt) nutzen eine Vielzahl an Tools und Anwendungen für ihre tägliche Arbeit. Diese unterschiedlichen Tools und Anwendungen stellen die User manchmal vor Herausforderungen im Gebrauch und somit benötigen die User schließlich Unterstützung von Supporteinheiten.

Hat ein User solch einen Supportfall, so kann dieser selbstständig über ein Dataport internes Selfservice Portal den Supportfall in ein Formular eingeben. In dem Formular werden bestimmte Fragen dem User zu seinem Supportfall gestellt … Anschließend wird das Formular eingereicht. Das Formular wird in ein Dataport internes Ticketsystem (ITSM) überführt, wodurch die First Supporteinheit per E‑Mail informiert wird. … Falls die First Supporteinheit keine Möglichkeit der Abhilfe hat, wird der Second Supporteinheit das Ticket übergeben. Die Second Supporteinheit verschafft dem User Abhilfe und informiert per E‑Mail den User.

Jederzeit hat der User die Möglichkeit über das interne Selfservice Portal den aktuellen Status seines Tickets einzusehen.

BPMN-Modell 1.0.0 – Kernprozess Supportfall

Beschreibung

Das Modell 1.0.0 bildet den Kernprozess eines Supportfalls ab: vom Erfassen der Störung über die Bearbeitung im 1st/2nd-Level bis zur Benachrichtigung und dem Schließen des Incidents.

Ablauf (Hauptschritte)

  1. User-Pool: Nutzer erstellt einen Supportfall und übermittelt ein Formular (Kurz-/Langbeschreibung, Rückrufoption etc.).
  2. ITSM-Pool: Ticketanlage im ITSM-System und Weiterleitung an den First Support.
  3. First Support: prüft den Fall.
    • Wenn lösbar: Lösung bereitstellen & dokumentieren, Nutzer informieren.
    • Wenn nicht lösbar: Ticket an Second Support eskalieren.
  4. Second Support: übernimmt, analysiert und behebt — inkl. Dokumentation.
  5. E-Mail-Benachrichtigungen: Nutzer wird über Status & Abschluss informiert.

Charakteristik

  • Fokus auf Standardfluss ohne erweiterte Sonderpfade.
  • Kompakte Darstellung für nachvollziehbare End-to-End-Transparenz.

BPMN-Modell 2.0.0 – Erweiterter Supportprozess

Beschreibung

Das Modell 2.0.0 ist eine erweiterte Version mit zusätzlichen Details und Qualitätssicherungen (z. B. Authentifizierung, Wissensdatenbank-Abfragen, Feedback-Schleifen und Timer-Events).

Ablauf (Hauptschritte & Erweiterungen)

  1. User-Pool: Login/Authentifizierung im Selfservice-Portal; Formular inkl. Abbruchpfade.
  2. ITSM-Pool: Ticketanlage mit Validierungen (z. B. Abgleich mit ITSM-Datenbank/DBSM).
  3. First Support: enthält zusätzliche Schritte
    • Wissensdatenbank-Abfrage (DBSM) zur schnelleren Lösungssuche.
    • Feedback-Schleife (z. B. Rückfragen oder Bestätigung durch den Nutzer).
  4. Second Support: analog zu 1.0.0, jedoch detaillierter dokumentiert und näher an ITIL-Standards (Eskalations- und Nachdokumentationspfade).
  5. User-Rückmeldungen: kurzes Feedback („Lösung erfolgreich ja/nein“).
  6. Timer-Events: zeitgesteuerte Aktionen (z. B. automatisches Schließen nach 30 Tagen).

Wesentliche Unterschiede zu 1.0.0

  • Vorangestellte Authentifizierung im Portal & valide Ticketanlage.
  • DBSM/Wissensdatenbank als Entscheidungshilfe im 1st-Level.
  • Feedback-Schleife mit Nutzerinteraktion, bevor geschlossen wird.
  • Timer-Events für SLA-/Prozess-Automatisierung.
